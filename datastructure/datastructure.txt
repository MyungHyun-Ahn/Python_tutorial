자료구조
데이터의 효율적인 접근 및 조작을 가능하게 해주는 저장 및 관리 방식

자료구조의 목적
자료를 구조화 > 데이터를 효율적으로 사용
컴퓨터에 데이터가 어떻게 저장되는지 알아야 됨!

스토리지 
- 데이터가 영구적으로 저장되는 곳
- 데이터를 많이 저장할 수 있음
- 데이터를 저장 / 받아오는데 오래걸림
> 오래보관할 데이터를 저장

# 자료구조에서는 메모리가 중요
메모리 : 램 = 메모리
- 데이터가 임시로 저장되는 곳
- 데이터를 저장하는 용량이 작음
- 데이터를 저장 / 받아오는데 빠름
> 지금 당장 사용해야하는 데이터를 저장

영화파일을 켜면
스토리지 > 메모리로 다운받고
메모리에서 재생함.

메모리
일정한 칸으로 나눠져 있다.
각 칸에 데이터를 저장할 수 있다.
각 칸은 자신만의 주소가 있다.

RAM : 임의 접근 메모리
임의 접근 : 저장 위치를 알면 접근할 때 항상 일정한 시간이 걸림 : 시간 복잡도 O(1)
순차 접근 : 저장된 위치까지 가는데 한 단계씩 거쳐야함.
임의 접근이 순차 접근보다 효율적임.

자료구조
메모리에 데이터를 저장한다.
메모리에 저장된 데이터를 찾는다.
> 임의 접근으로 하고 있다!! O(1)

레퍼런스(reference)
- 데이터에 접근할 수 있게 해주는 값
- '주소'보다 조금 더 포괄적인 표현
- 자료구조를 배울 때는 "주소 = 레퍼런스"

변수를 사용할 때는 저장된 값을 알아서 받아옴

데이터의 주소
파이썬 id 함수 : 저장한 데이터의 메모리 저장 위치를 받아온다.
같은 주소에 저장되어있는 데이터는 똑같은 데이터이다.
